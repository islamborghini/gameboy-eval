# gameboy-eval — Design Doc (rev. 3)

*An open-source benchmark that measures how well coding agents can build an emulator from
scratch — graded automatically, deterministically, and for free on a laptop.*

> Working name: **gameboy-eval**. First target system: **Game Boy (DMG)**.
> **Rev. 3** re-aims the project at *maximum fidelity* to Mechanize's GBA Eval, adapted to
> the Game Boy. It drops rev. 2's "Python SM83 core + stdin/stdout JSON vectors + single-shot"
> as the *product*: the graded artifact is now a full emulator (Rust → WASM), graded by a
> replay-led composite against a black-box **SameBoy oracle**, generated by an **agentic,
> offline** environment. Disciplined staging is kept; every milestone now builds toward the
> faithful architecture rather than a divergent shortcut.

---

## 1. Pitch

Give a coding agent the task "write a Game Boy emulator from scratch," then grade what it
produces automatically and deterministically — by replaying recorded inputs against a
known-accurate reference emulator and scoring how *visually close* the result stays, frame
by frame, plus standard self-checking test ROMs and an audio spectral comparison. Free,
local-first, runs on a MacBook.

## 2. Load-bearing principles

1. **Determinism makes grading possible.** The Game Boy has no entropy source. Identical
   inputs → bit-identical output. We exploit this everywhere.
2. **Generation and grading are separate phases.** The agent produces an artifact (a Rust
   → `wasm32-unknown-unknown` module). We save it and grade later, offline, model out of
   the loop — free, reproducible, provider-independent.
3. **Quality-first grading.** Many small, unambiguous checks over one fuzzy one.
4. **Grade for "almost the same almost everywhere," never exact-match.** Even excellent
   emulators diverge from the reference on incidental timing/rendering a human never
   notices. The score reflects *human-perceived playability*, not bit-equality (see §3.4).
5. **Fairness = a senior engineer could ace it in the same environment.** If a senior
   systems engineer dropped into the identical container with the identical tools couldn't
   reach ~1.0, the eval is measuring resourcefulness, not the capability we care about.
6. **Watch measurement validity, not just fairness.** Keep asking whether a high score
   actually reflects "can build an emulator" and not some incidental workaround skill.

## 3. Architecture (the faithful target)

### 3.1 Candidate artifact + ABI
A Rust cargo project (package `gb_emu`) compiling to a `wasm32-unknown-unknown` **cdylib**
that exports the lockstep ABI in [`spec/ABI.md`](spec/ABI.md): `alloc`/`dealloc`, `init`,
`load_boot_rom`, `load_rom`, `reset`, `set_keys`, `run_frame`, `framebuffer`, `audio`/
`audio_len`. Enforced constraints: fixed artifact path
`target/wasm32-unknown-unknown/release/gb_emu.wasm`, fixed package name, no
`.cargo/config.toml`, no custom `RUSTFLAGS`, build is exactly
`cargo build --release --lib --target wasm32-unknown-unknown`. **Grading drives the WASM
module via `wasmtime`** — the WASM runtime is the source of truth (and the same artifact is
browser-runnable for the future leaderboard).

### 3.2 Oracle (reference as a black box)
**SameBoy**, wrapped as an HTTP service in a *separate* container, exposed two ways shaped
exactly like the candidate ABI:
- batch: `run <rom> <frames> --replay … --dump-frames … --dump-audio …`
- stateful session: `set-keys` / `run-frame` / `framebuffer` / `audio` (one `run-frame` =
  one frame).
The agent reaches it only over HTTP via an `oracle` CLI client; **its source is never
exposed** (the agent has a shell and could decompile a co-located binary). Use SameBoy's
**open boot ROM** for both oracle and candidate; verify no divergence vs. a reference across
the suite before trusting it.

### 3.3 Generation environment (agentic, offline, continuous runtime)
The model runs **inside** a container with shell access, its Rust project at `/task/`, the
`oracle` client, `dev-roms/` (homebrew, excluded from the test set), Pan Docs, and the boot
ROM. **No internet** except the agent CLI's own model-provider endpoint via a proxy
allowlist (Game Boy is heavily tutorialized — without this the model just fetches SameBoy or
a tutorial and copies). **Continuous runtime:** the task never ends; if the agent tries to
stop it is relaunched with its working tree intact. No hints about internal architecture —
only the artifact contract is fixed. See [`env/TASK.md`](env/TASK.md).

### 3.4 Grading: composite + replay SSIM pipeline
```
overall = 0.60 · replay  +  0.20 · audio  +  0.20 · procedural
```
- **procedural** — open self-checking GB test ROMs (`c-sp/game-boy-test-roms`: Blargg,
  Mooneye, dmg/cgb-acid2, Mealybug) scored on final state (`LD B,B` → Fibonacci registers
  `B=3,C=5,D=8,E=13,H=21,L=34`, or framebuffer-matches-reference-screenshot for image tests).
- **replay** (centerpiece) — candidate and oracle run in **lockstep** on recorded inputs;
  per-frame structural-similarity, averaged. **Never** frames-until-first-divergence.
- **audio** — log-mel spectrogram distance through the same per-frame-Hill-then-mean pipeline.

**Replay per-frame score:** `score(d, τ) = 1 / (1 + (d/τ)^4)`; `replay = mean_t score(d_t, τ)`.
Pipeline (GBA-Eval validated defaults — start here, don't re-derive):
1. **Lockstep capture** via `set_keys`/`run_frame`/`framebuffer`. A replay is a text file of
   `<frame> <keys_hex>` lines driving both emulators.
2. **Color normalize:** quantize each channel to 5 bits → expand to 8 → BT.601 luma
   `Y = 0.299R + 0.587G + 0.114B`; compare on luma. (For DMG this is nearly a no-op — 4 greys
   — and only becomes load-bearing for GBC's 15-bit color. One reason DMG is the right first
   target.)
3. **Block SSIM:** non-overlapping blocks, classical SSIM per block, `defect = 1 − SSIM`.
   GBA validated ~10×10 on 240×160; the GB screen is **160×144**, so default to **8×8**
   (a 20×18 grid); treat block size as a sweepable parameter.
4. **Per-frame defect:** per-block **perceptual floor 0.15** (blocks below → contribute 0),
   then mean of floored block defects.
5. **Blank-frame gate:** candidate ≥99.9% one color while the reference isn't → defect 1.0.
6. **Adaptive τ:** SSIM-defect between *consecutive reference frames* (only moving frames),
   **90th percentile**, clamp **[0.005, 0.35]**.
7. **Hill + mean:** `1/(1+(d/τ)^4)` per frame (exponent 4), arithmetic mean over frames.

Why single-scale SSIM (not GMSD / pixel-proportion / fixed-τ / MS-SSIM): pixel-proportion
lets a blank screen score ~99%; fixed-τ can't be fair on flashing intros *and* discriminating
on calm scenes; GMSD's variance pooling can't separate harmless scattered noise from one big
concentrated defect; there's no sub-native detail in pixel art for MS-SSIM to use.

**Cost discipline:** WASM emulators run <10× realtime → keep replays short (seconds to a
couple of minutes), targeting milestones ("clears level 1"), never full playthroughs.

### 3.5 Score bands (apply to the composite; reference-vs-itself = 1.00)
| Band | ~score | Looks like |
|------|--------|-----------|
| Doesn't run | ~0–5% | title screen at best, or won't load |
| Barely works | ~15–30% | loads, breaks as real play starts |
| Plays incorrectly | ~45–55% | mostly right, timing drift → eventual permanent desync |
| Mostly playable | ~70% | plays correctly; cosmetic glitches |
| Near-reference | ~85–99% | indistinguishable-to-near-indistinguishable |

## 4. Build order (status)

- [x] **M0** Scaffold: repo layout, Python 3.12 venv, NOTICE, ABI/TASK/design docs.
- [x] **M1** Toolchain: Rust 1.96 + `wasm32-unknown-unknown` + wasmtime 45 + Docker;
  hello-world cdylib → `.wasm` → callable (incl. linear-memory read) via wasmtime.
- [x] **M2** SameBoy driven as a black-box **oracle** via its libretro core (in-process for
  now; HTTP wrapper deferred to M5). Gate passed: dmg-acid2 renders, deterministic, DMG-forced.
- [x] **M3** Lockstep harness + reference candidate — BOTH gates passed (metric validated on
  oracle-vs-perturbed; rboy-wasm scores 1.0000 vs the oracle):
  - (a) validate the SSIM/lockstep math with **oracle-vs-oracle (≈1.00)** and
    **oracle-vs-deliberately-perturbed (low / blank-gated)** — proves the grader detects
    failure, not just self-agreement;
  - (b) bring up the real WASM candidate path by **porting a mature open-source Rust GB
    emulator** to the ABI, compiling to `.wasm`, driven by a wasmtime runner in lockstep
    with the oracle. Gate: that known-good candidate scores **≈1.0** vs the oracle.
- [ ] **M4** Full grader: replay SSIM + procedural test-ROM section + audio + composite +
  score-band report. Gate: composite computed; reference ≈1.0, broken ≈0.
- [ ] **M5** Agentic generation environment: `--network none` container with shell, Rust
  project, `oracle` CLI, `dev-roms/`, Pan Docs, boot ROM, continuous-runtime `TASK.md`,
  agent loop behind a provider-only proxy. Run a model; **expect ~0** for a small local
  model. Success = the environment runs end-to-end and is fair.
- [ ] **M6** Audio polish + leaderboard with in-browser WASM artifacts.

**Spine:** validate the grading harness against a **known-good emulator (M2–M4) before**
building the generation side (M5). Same "grader against itself" logic as rev. 2, lifted to
the full-emulator level — de-risking the SSIM pipeline before any model is involved.

## 5. Decisions (resolved)
- **Language:** Rust → WASM is the product (browser-runnable + sandboxed). An optional
  internal SM83 JSON-vector smoke-test (flat-memory, stdin/stdout) is kept only for
  *debugging the reference candidate's CPU*, never as the graded product.
- **Generation:** agentic + continuous-runtime + offline (not single-shot). Provider layer
  swappable (Ollama `qwen2.5-coder:7b` locally — expected ~0 on the full task — or a capable
  API model for the occasional real signal).
- **DMG first, GBC later** (keeps color-normalization moot until it matters).
- **Strictness:** SSIM "almost-same-everywhere", not exact-match; procedural exact on final
  state.
- **Integrity:** keep a private held-out split of grading ROMs/recordings (the harness is
  open); `dev-roms/` shown to the model are excluded from grading.

## 6. Changelog
- **rev. 1:** CPU-centric; exact-match replay tier (later shown to score everything ~0).
- **rev. 2:** adopted GBA-Eval lessons (SSIM "almost-same-everywhere", oracle, offline
  generation, held-out split); but kept a Python-core + JSON-vector + single-shot v1.
- **rev. 3 (this doc):** re-aimed to maximum fidelity — full-emulator Rust→WASM artifact,
  replay-led composite vs. a SameBoy oracle, agentic offline generation. Grader validated
  against a known-good emulator before any model is involved.
